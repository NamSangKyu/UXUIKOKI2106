
	절차지향언어
		일을하는 순서에 초점을 맞춘 언어 --> C, 어셈블리 ......
		
	객체지향언어
		일을 하는 대상을 표현(코드)
		
		1. 캡슐화(Encapsulation)
		
		2. 상속(Inheritance)
		
		3. 다형성(Polymorphism)
	
	클래스
		표현할려는 대상을 표준화하여 코드로 작성한 것을 클래스
		클래스 구성 요소
			필드 : 저장할 데이터를 담당, 변수선언
			메서드 : 기능적인 부분을 담당, 메서드 선언
			생성자 : 객체 생성시 해야되는 첫번째 기능
			
	접근제어자
		private : 해당 클래스 내부에서만 접근 가능 
		protected : 상속받은 클래스까지 접근이 가능
		default : 같은 패키지 내에 있으면 접근이 가능
		public : 외부에서도 접근이 가능
