try : 작업영역, 대신에 이 영역에선 Exception 발생할수 있다.
catch : 발생한 Exception을 처리하는 영역
finally : 작업이 정상적으로 완료가 되거나, Exception 발생하더라도 반드시 실행되는 영역
		  무조건 실행되는 영역
throw : 강제로 Exception 발생
throws : Exception 발생시 호출한 메서드로 Exception을 넘김

사용자 정의 Exception
	프로젝트에서 작성하던 기능들 중 사용자가 필요한 Exception 작성
	Exception 클래스를 상속받으면 Exception 클래스가 됨
		- Exception message 만 생성자로 받으면 됨
